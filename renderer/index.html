PROXY SETUP
To route OBSIDIAN scan traffic through Burp:
1. Set Burp proxy listener to 127.0.0.1:8080
2. Add a proxy in OBSIDIAN's Proxy Manager:
   HTTP / 127.0.0.1 / 8080
3. Launch scan — all HTTP traffic routes through Burp
CERTIFICATE
For HTTPS interception, install the Burp CA cert:
1. Navigate to http://burpsuite in a browser
2. Download the CA certificate
3. Install it as a trusted root authority
4. Enable SSL passthrough for tools that pin certificates

WINDOWS DETECTED
WSL was not detected on this machine.
The scan tools (nmap, nuclei, nikto etc.) require Linux.

Fix in 2 steps:
1. Open PowerShell as Administrator and run:
   wsl --install
2. Restart your machine, then re-open OBSIDIAN.

OBSIDIAN will automatically detect WSL and route all
scan traffic through it — no extra configuration needed.

The UI, exports, proxy manager and script generator
all work normally on Windows without WSL.
